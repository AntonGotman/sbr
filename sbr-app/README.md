# СБР — Сервис бронирования ресурсов

Фронтенд на Vue 3 + Vite + Pinia + Vue Router, повторяющий дизайн из приложенного макета
(шрифт Nunito Sans, палитра `#43716B` / `#86AEAA` / `#DCE7E6` / `#F2F2F2`).

## Запуск

```bash
npm install
npm run dev
```

Приложение откроется на `http://localhost:5173`.

Продакшн-сборка:

```bash
npm run build
npm run preview
```

## Демо-вход

Авторизация имитируется на фронтенде (без backend): на странице **Вход** подойдёт
любой email и пароль, на странице **Регистрация** — имя, email и пароль.

## Что реализовано

- **Лендинг** — герой с названием сервиса, кнопками «Вход» / «Регистрация» и блоком возможностей.
- **Пространство** — список бронируемых пространств (коворкинг, переговорная) с расписанием
  на выбранный день, поиском, сортировкой и добавлением новых пространств.
- **Аппаратура** — отдельный список бронируемого оборудования с возможностью добавления и удаления.
- **Люди** — каталог коллег/подрядчиков с поиском, сортировкой, добавлением новых карточек
  и кнопкой «Пригласить».
- **Бронирование** — модальное окно с выбором названия мероприятия, временных слотов,
  ответственных лиц и комментария.
- **Защита от повторного бронирования** — при пересечении выбранных слотов с уже занятыми
  показывается окно ошибки с объяснением, какое мероприятие занимает это время.
- **Уведомления** — всплывающие тосты о создании бронирования, приглашении человека,
  принятии/отклонении приглашения.
- **Активные приглашения** — карточки в сайдбаре с возможностью принять/отклонить.
- **Календарь** — навигация по месяцам, выбор даты, выходные дни выделены.
- **Профиль** — выпадающее меню с именем, email и выходом из аккаунта.

## Структура проекта

```
src/
  components/
    layout/     — хедер, меню профиля
    ui/          — модалки, тосты, грид слотов, календарь
    resources/   — карточка ресурса, модалка бронирования, карточка приглашения
    people/      — карточка человека
  store/         — Pinia: auth, resources, people, invitations, ui, constants
  views/         — страницы (Landing, Login, Register, Space, People, Equipment)
  router/        — маршруты и guard на авторизацию
```

## Данные

Все данные (ресурсы, бронирования, люди, приглашения) хранятся в памяти приложения
(Pinia store) и сбрасываются при обновлении страницы — это фронтенд-прототип без backend.
Для продакшна стоит заменить действия в `store/resources.js`, `store/people.js`,
`store/invitations.js` и `store/auth.js` на запросы к реальному API.
